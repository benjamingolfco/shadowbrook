# E2E Tests Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Set up Playwright e2e testing infrastructure and write one happy-path test for the walkup waitlist flow.

**Architecture:** Playwright installed in `src/web/` with tests in `src/web/e2e/`. Tests run against a configurable base URL (test environment or localhost). Auth abstraction via custom fixture, page objects for readability.

**Tech Stack:** Playwright, TypeScript, pnpm

**Spec:** `docs/superpowers/specs/2026-03-24-e2e-tests-design.md`

```typescript
import { test as base, type Page } from '@playwright/test';

/**
 * Auth fixture that abstracts authentication.
 * Currently uses the dev role switcher.
 * When real auth lands, swap the internals without changing test files.
 */

async function setRole(page: Page, role: 'golfer' | 'operator' | 'admin') {
  // The dev role switcher stores the role in localStorage.
  // Set it directly to avoid UI interaction overhead.
  await page.evaluate((r) => localStorage.setItem('shadowbrook-dev-role', r), role);
}

export const test = base.extend<{
  asGolfer: () => Promise<void>;
  asOperator: () => Promise<void>;
  asAdmin: () => Promise<void>;
}>({
  asGolfer: async ({ page }, use) => {
    await use(async () => {
      await setRole(page, 'golfer');
    });
  },
  asOperator: async ({ page }, use) => {
    await use(async () => {
      await setRole(page, 'operator');
    });
  },
  asAdmin: async ({ page }, use) => {
    await use(async () => {
      await setRole(page, 'admin');
    });
  },
});

export { expect } from '@playwright/test';
```

Note: The `setRole` implementation uses the `shadowbrook-dev-role` localStorage key, matching `MockAuthProvider.tsx`. The walkup flow (`/join`) does not require auth, so this fixture won't be used by the walkup test — but it's needed for future flows.

### Task 3: Create walkup page object

**Files:**
- Create: `src/web/e2e/fixtures/walkup-page.ts`

- [ ] **Step 1: Create the page object**

Create `src/web/e2e/fixtures/walkup-page.ts`:

```typescript
import { type Page, type Locator } from '@playwright/test';

export class WalkupPage {
  private readonly codeInput: Locator;
  private readonly firstNameInput: Locator;
  private readonly lastNameInput: Locator;
  private readonly phoneInput: Locator;
  private readonly joinButton: Locator;

  constructor(private readonly page: Page) {
    this.codeInput = page.getByLabel('4-digit course code');
    this.firstNameInput = page.getByLabel('First Name');
    this.lastNameInput = page.getByLabel('Last Name');
    this.phoneInput = page.getByLabel('Phone Number');
    this.joinButton = page.getByRole('button', { name: 'Join Waitlist' });
  }

  async goto() {
    await this.page.goto('/join');
  }

  async enterCode(code: string) {
    await this.codeInput.fill(code);
  }

  async fillJoinForm(data: { firstName: string; lastName: string; phone: string }) {
    await this.firstNameInput.fill(data.firstName);
    await this.lastNameInput.fill(data.lastName);
    await this.phoneInput.fill(data.phone);
    await this.joinButton.click();
  }

  getConfirmationHeading() {
    return this.page.getByRole('heading', { name: /You're on the list/ });
  }

  getPositionText() {
    return this.page.getByText(/#\d+ in line at/);
  }
}
```

Selectors are derived from the actual component markup:
- Code input: `aria-label="4-digit course code"` (from `CodeEntry.tsx:75`)
- Form fields: `<FormLabel>` text (from `JoinForm.tsx:94,108,122`)
- Join button: button with text "Join Waitlist" (from `JoinForm.tsx:148`)
- Confirmation heading: "You're on the list, {name}!" (from `Confirmation.tsx:31`)
- Position text: "#{position} in line at {course}" (from `Confirmation.tsx:33`)

### Task 4: Write the walkup happy path e2e test

**Files:**
- Create: `src/web/e2e/tests/walkup/walkup-join.spec.ts`

- [ ] **Step 1: Write the test**

Create `src/web/e2e/tests/walkup/walkup-join.spec.ts`:

```typescript
import { test, expect } from '@playwright/test';
import { WalkupPage } from '../../fixtures/walkup-page';
import { TEST_WALKUP_CODE, TEST_GOLFER } from '../../fixtures/test-data';

test.describe('Walkup Join Flow', () => {
  test('golfer can join the waitlist via walkup code', async ({ page }) => {
    const walkup = new WalkupPage(page);

    await walkup.goto();
    await walkup.enterCode(TEST_WALKUP_CODE);

    // After valid code, the join form should appear
    await expect(page.getByLabel('First Name')).toBeVisible();

    await walkup.fillJoinForm(TEST_GOLFER);

    // Should see confirmation with position
    await expect(walkup.getConfirmationHeading()).toBeVisible();
    await expect(walkup.getPositionText()).toBeVisible();
  });
});
```

- [ ] **Step 1: Create the workflow file**

Create `.github/workflows/e2e.yml`:

```yaml
name: E2E Tests

on:
  workflow_dispatch:
  # Uncomment when deploy workflow exists:
  # workflow_run:
  #   workflows: ["Deploy to Test"]
  #   types: [completed]
  #   branches: [main]

env:
  E2E_BASE_URL: https://test.shadowbrook.golf

jobs:
  e2e:
    name: Playwright E2E
    runs-on: ubuntu-latest
    # Uncomment when using workflow_run:
    # if: ${{ github.event.workflow_run.conclusion == 'success' }}
    steps:
      - uses: actions/checkout@v4

      - uses: pnpm/action-setup@v4

      - uses: actions/setup-node@v4
        with:
          node-version: 22
          cache: pnpm
          cache-dependency-path: src/web/pnpm-lock.yaml

      - name: Install dependencies
        run: pnpm --dir src/web install --frozen-lockfile

      - name: Install Playwright Chromium
        run: pnpm --dir src/web exec playwright install --with-deps chromium

      - name: Run E2E tests
        run: pnpm --dir src/web e2e

      - name: Upload test report
        uses: actions/upload-artifact@v4
        if: ${{ !cancelled() }}
        with:
          name: playwright-report
          path: src/web/e2e/playwright-report/
          retention-days: 14

      - name: Upload trace artifacts
        uses: actions/upload-artifact@v4
        if: failure()
        with:
          name: playwright-traces
          path: src/web/e2e/test-results/
          retention-days: 14
```

The `workflow_run` trigger is commented out since the deploy workflow doesn't exist yet. For now, the workflow can be triggered manually via `workflow_dispatch`. Uncomment the `workflow_run` section when the deploy-to-test workflow is in place.
